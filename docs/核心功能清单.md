# xAgt 核心功能清单

> 项目类型：**OpenCode TUI 多 Agent 工作系统插件**（TypeScript，Node.js 运行时）
> 技术栈：TypeScript / @opencode-ai/plugin / Node.js

---

## 板块一：Agent 角色体系

| # | 功能 | 说明 | 关键文件 |
|---|------|------|---------|
| 1 | **五 Agent 角色定义** | Vox（总指挥）、Lynx（侦察兵）、Fixer（执行者）、Judge（审计员）、Smith（元审查），各有独立 system prompt 和工具权限 | `src/agents/*.ts` |
| 2 | **Agent 注册中心** | 统一的 Agent 注册与名称类型导出，供网关和 hooks 引用 | `src/agents/index.ts` |

## 板块二：安全网关

| # | 功能 | 说明 | 关键文件 |
|---|------|------|---------|
| 3 | **工具权限策略** | 声明式的 Agent 工具白名单/黑名单（如 Vox 禁止直接 read/write/glob） | `src/gateway/policy.ts` |
| 4 | **运行时拦截器** | 权限校验 + 危险命令检测（拦截越权工具调用） | `src/gateway/interceptor.ts` |
| 5 | **Session-Agent 映射** | 将 OpenCode sessionID 反向解析为 Agent 名称，解决身份识别问题 | `src/gateway/session-registry.ts` |

## 板块三：记忆系统

| # | 功能 | 说明 | 关键文件 |
|---|------|------|---------|
| 6 | **JSONL 持久化存储** | 零依赖的 JSONL 存储引擎，自动轮转（上限 200 条）、防并发写入、防文件损坏 | `src/memory/store.ts` |
| 7 | **三级记忆注入** | 从记忆中提取 lessons / patterns / decisions 三类信息注入 Vox system prompt | `src/memory/hierarchy.ts` |
| 8 | **会话上下文轮转** | 会话压缩时智能提取经验教训，保存到记忆库 | `src/memory/context-rollover.ts` |

## 板块四：任务编排与看板

| # | 功能 | 说明 | 关键文件 |
|---|------|------|---------|
| 9 | **后台任务看板** | 跟踪 `task()` 调用生命周期（运行中/已完成），自动注入到用户消息尾部 | `src/utils/background-job-board.ts` |
| 10 | **断点续传** | Fixer 任务状态持久化到 `.xagt/task_state.json`，进程中断后可恢复 | `src/hooks/task-state-persistence.ts` |
| 11 | **系统约束强制注入** | 每轮对话向 Vox 注入 6 条不可绕过的硬约束 | `src/hooks/system-transform.ts` |

## 板块五：成本优化

| # | 功能 | 说明 | 关键文件 |
|---|------|------|---------|
| 12 | **LRU 工具缓存** | 缓存只读 Agent 的幂等工具调用结果，支持 TTL 过期和写操作前缀失效 | `src/cost/cache.ts` |
| 13 | **模型路由建议** | 按 Agent 角色推荐性价比模型（performance / balanced / economy 三级） | `src/cost/router.ts` |

## 板块六：数据分析

| # | 功能 | 说明 | 关键文件 |
|---|------|------|---------|
| 14 | **失败事件采集** | 采集 Judge 拒绝 + Fixer 失败事件，Smith 每 30 轮生成分析报告 | `src/analytics/` |

## 辅助设施

| 功能 | 说明 | 关键文件 |
|------|------|---------|
| **Git 沙盒** | 为 Fixer 创建独立 Git Worktree，修改后合并回主线，含冲突安全回退 | `src/git-sandbox.ts` |
| **Smith 频率触发器** | 按 session 隔离计数器，每 30 轮用户消息激活 Smith | `src/hooks/smith-trigger.ts` |
| **Vox 输出拦截** | 检测 Vox 回复含代码但没有 `task()` 时自动拦截并替换为警告 | `src/index.ts` |
| **错误码系统** | `FE::ERR::E5xxx` 体系 | `docs/error-codes.md` |

## 架构数据流

```
用户输入
  │
  ▼
Vox（总指挥）
  ├── task("lynx", ...)  ──→ Lynx（只读调研） → 结果回 Vox
  ├── task("fixer", ...) ──→ Fixer（改代码） → Judge（审查） → 通过则合并
  ├── task("judge", ...) ──→ Judge（审查）   → 通过/拒绝报告
  └── task("smith", ...) ──→ Smith（每30轮元审查） → 分析报告
        │
        └─── 所有工具调用必经 ──→ Tool Gateway（权限校验）
                                      │
                                      └─── MemoryStore（记忆持久化）
                                      └─── BackgroundJobBoard（看板跟踪）
                                      └─── ToolResultCache（成本优化）
```
