# [07-05][前端]Vox 派活提示词优化

## 状态

- 待解决

## 优先级

- 中

## 类别

- [前端：代码冗余]

## 问题描述

- Vox 在 task() 中对子代理（Fixer/Lynx/Judge）的 prompt 过于冗长，重复了子代理 system prompt 中已有的铁律和规则
- 当前模式：task("fixer", "按照铁律1-4...记得跑测试...不要擅自扩大... 【实际任务指令】")
- 优化目标：task("fixer", "【文件】路径 | 【操作】具体修改 | 【验收】命令"）
- 影响所有子代理：Fixer、Lynx、Judge

## 影响范围

- Vox 调度子代理的提示词质量，影响 deepseek-chat 模型的执行准确率

## 方案建议

1. 修改 src/agents/vox.ts 中的调度规则，增加三段式模板
2. 为每个子代理类型（Fixer/Lynx/Judge）定义固定模板
3. 模板化：
   - Fixer：`【文件】|【操作】|【验收】`
   - Lynx：`【目标】|【线索】|【期望】`
   - Judge：`【审查】|【重点】|【标准】`
