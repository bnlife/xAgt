大模型（DeepSeek）只是“大脑”，而 Agent Runtime 就是你要搭建的“身体”和“社会规则”。它负责接收大脑的指令、限制大脑的冲动、执行物理操作、并反馈结果。
以下是你要做一个完整的 Agent Runtime 具体需要做的事和落地步骤：
一、Agent Runtime 具体能做什么？（核心模块）
你的 Runtime 本质上是一个中间件，它包裹在 LLM API 之上，提供以下 4 大核心能力：
上下文管理器：不让 AI 患“失忆症”。自动注入项目状态、任务进度、历史教训；在上下文超限时自动压缩。
工具路由与权限网关：不让 AI “乱动手”。拦截 AI 的工具调用请求，根据角色（Vox/Fixer/Judge）校验权限，并在安全环境中执行。
编排与状态机：不让 AI “短路”。解析主控的调度指令，拉起子 Agent，等待回报，强制走 Judge 审查闭环。
沙盒与版本控制：不让 AI “炸机房”。自动管理 Git 分支，限制 bash 命令的破坏力。

第二步：实现工具调用拦截器
这是你 Hook 的升级版。每次 LLM 试图调用工具时，必须经过这一层。
// tool_gateway.ts
async function executeTool(agentRole: string, toolName: string, params: any) {
  // 1. 权限校验
  if (!AgentConfig[agentRole].allowedTools.includes(toolName)) {
    return { error: `权限拒绝：${agentRole} 无权调用 ${toolName}` };
  }
  // 2. 危险操作拦截 (针对 Fixer)
  if (toolName === 'run_bash') {
    const cmd = params.command;
    if (cmd.includes('rm -rf') || cmd.includes('drop database')) {
      return { error: "安全拦截：禁止执行破坏性命令" };
    }
  }
  // 3. 执行真实操作
  try {
    switch(toolName) {
      case 'write_file': return await fs.writeFile(params.path, params.content);
      case 'run_bash': return await execBashInSandbox(params.command); // 在沙盒里跑
      // ... 其他工具
    }
  } catch (e) {
    return { error: e.message };
  }
}
第三步：实现编排引擎
解析 Vox 的 task() 指令，拉起子 Agent。这是实现多智能体协作的核心。
// orchestrator.ts
async function runAgent(role: string, userInput: string, parentContext: any) {
  const config = AgentConfig[role];
  const messages = [
    { role: "system", content: config.systemPrompt },
    { role: "user", content: userInput }
  ];
  while (true) {
    // 1. 调用大模型
    const llmResponse = await callDeepSeek(config.model, messages, config.allowedTools);
    // 2. 如果模型决定不调工具，直接返回最终文本
    if (!llmResponse.tool_calls) {
      return llmResponse.content;
    }
    // 3. 如果模型要调工具，进入工具执行循环
    for (const toolCall of llmResponse.tool_calls) {
      let result;
      // 特殊路由：如果是 task 工具，说明是 Vox 在调度子 Agent
      if (toolCall.name === 'task') {
        const { subagent_type, prompt } = JSON.parse(toolCall.arguments);
        // 递归调用 runAgent 拉起子 Agent
        result = await runAgent(subagent_type, prompt, messages);
      } else {
        // 普通工具，走网关执行
        result = await executeTool(role, toolCall.name, toolCall.arguments);
      }
      // 将工具结果塞回上下文，让模型继续思考
      messages.push({ role: "tool", tool_call_id: toolCall.id, content: JSON.stringify(result) });
    }
  }
}

// memory_injector.ts
function prepareSystemPrompt(role: string) {
  let basePrompt = AgentConfig[role].systemPrompt;
  if (role === 'vox') {
    // 自动读取并注入当前状态
    const taskState = fs.readFileSync('docs/.task_state.json', 'utf-8');
    const lessons = fs.readFileSync('docs/lessons_learned.md', 'utf-8').slice(0, 1000);
    basePrompt += `\n\n[系统注入的实时记忆]
    当前任务状态: ${taskState}
    历史教训: ${lessons}
    `;
  }
  return basePrompt;
}
第五步：实现 Git 分支沙盒管理
在 Vox 进入 S5 阶段前，Runtime 自动创建分支；在 S6 阶段，自动合并。
// git_sandbox.ts
async function setupSandbox(taskId: string) {
  const branchName = `ai/task-${taskId}`;
  await exec(`git checkout -b ${branchName}`);
  console.log(`[Runtime] 已进入沙盒分支: ${branchName}`);
  return branchName;
}
async function commitAndMerge(branchName: string, message: string) {
  await exec(`git add .`);
  await exec(`git commit -m "${message}"`);
  await exec(`git checkout main`);
  await exec(`git merge ${branchName}`);
  await exec(`git branch -d ${branchName}`);
  console.log(`[Runtime] 沙盒任务完成，已合并至 main`);
}
三、你的 Runtime 运行时的全景工作流
当你把上面的模块组合起来，你的开发体验会变成这样：
你输入："加一个用户登录页面，用 Tauri invoke 传数据"。
Runtime 启动 Vox：
注入器自动把 架构地图.md 和 .task_state.json 塞给 Vox。
Vox 思考后，输出：task("lynx", "去查 Tauri invoke 的用法")。
Runtime 拦截到 task：
发现是调度 Lynx，立刻拉起一个 Lynx 专用的 LLM 实例。
Lynx 跑完，把结果（一段文档）返回给 Runtime。
Runtime 把结果塞回 Vox 的上下文。
Vox 继续思考：输出 task("fixer", "在 src/api/auth.ts 实现登录逻辑")。
Runtime 拦截到 task：
自动执行 git checkout -b ai/auth-login（进入物理沙盒）。
拉起 Fixer LLM 实例。
Fixer 试图调用 run_bash("npm install axios")，被网关拦截（越权安装依赖）。
Fixer 改用 edit_file 修改代码，成功。
Vox 收到 Fixer 完成报告：输出 task("judge", "审查 src/api/auth.ts")。
Runtime 拉起 Judge：Judge 读取代码，发现没打 RS:: 日志，返回 拒绝。
Vox 收到拒绝报告：再次调用 task("fixer", "补上日志前缀")。
循环往复，直到 Judge 返回 通过。
Vox 输出最终汇报："登录页已加完，测试通过"。
Runtime 自动执行：git commit & git merge main。清空沙盒。
