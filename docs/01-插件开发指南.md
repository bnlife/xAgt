> 基于 OpenCode 官方文档和社区实践整理
> 调研日期：2026-07-03

---

## 一、插件系统架构概述

OpenCode 插件系统是其核心扩展机制，允许第三方开发者通过 JavaScript/TypeScript 模块注入自定义工具、修改 LLM 调用参数、拦截消息、监听事件，甚至改变 Agent 的行为模式。

### 核心设计原则

- **函数式设计**：插件是一个异步函数，接收 `PluginInput` 上下文，返回 `Hooks` 对象
- **链式执行**：多个插件的 Hook 按注册顺序依次执行
- **共享输出**：所有插件共享同一个 `output` 对象引用，后续插件能看到前序插件的修改
- **容错优先**：Hook 中的错误不应导致 OpenCode 崩溃

### 插件加载来源（按优先级顺序）

1. 内置插件（INTERNAL_PLUGINS）
2. 全局配置 `~/.config/opencode/opencode.json` 中的 `plugin` 数组
3. 项目配置 `opencode.json` 中的 `plugin` 数组
4. 全局插件目录 `~/.config/opencode/plugins/`
5. 项目插件目录 `.opencode/plugins/`

## 二、创建插件项目

### 项目结构

```
my-opencode-plugin/
├── src/
│   └── index.ts          # 插件入口
├── package.json
├── tsconfig.json
└── README.md
```

### package.json 配置

```json
{
  "name": "opencode-my-plugin",
  "version": "0.1.0",
  "type": "module",
  "main": "./dist/index.js",
  "types": "./dist/index.d.ts",
  "files": ["dist", "README.md", "LICENSE"],
  "peerDependencies": {
    "@opencode-ai/plugin": "^1.0.0"
  },
  "devDependencies": {
    "typescript": "^5.0.0",
    "@types/node": "^20.0.0",
    "@opencode-ai/plugin": "^1.0.0"
  },
  "scripts": {
    "build": "tsc -p tsconfig.json",
    "prepublishOnly": "npm run build"
  }
}
```

### tsconfig.json 配置

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "declaration": true,
    "outDir": "./dist"
  },
  "include": ["."]
}
```

### 基本插件结构

```typescript
import type { Plugin } from "@opencode-ai/plugin"

export const MyPlugin: Plugin = async ({ project, client, $, directory, worktree }) => {
  console.log("Plugin initialized for:", project.name)

  return {
    // 在这里返回 Hook 实现
  }
}
```

### PluginInput 上下文对象

| 属性 | 描述 |
|------|------|
| `project` | 当前项目信息 |
| `directory` | 当前工作目录 |
| `worktree` | Git worktree 路径 |
| `client` | OpenCode SDK 客户端，用于与 AI 交互 |
| `$` | Bun 的 shell API，用于执行命令 |

## 三、Hook 系统详解

### 3.1 核心 Hook 类型

#### 事件监听 Hook

```typescript
event?: (input: { event: Event }) => Promise<void>
```

监听 OpenCode 中的各种事件，如 session 创建、消息更新、工具执行等。

#### 配置注入 Hook

```typescript
config?: (input: Config) => Promise<void>
```

在插件加载时修改配置，如设置默认 agent、注册命令等。

#### 自定义工具注册 Hook

```typescript
tool?: { [key: string]: ToolDefinition }
```

注册自定义工具，让 AI 可以调用。

#### 消息处理 Hook

```typescript
"chat.message"?: (
  input: {
    sessionID: string
    agent?: string
    model?: { providerID: string; modelID: string }
    messageID?: string
    variant?: string
  },
  output: { message: UserMessage; parts: Part[] }
) => Promise<void>
```

拦截和处理聊天消息。

#### LLM 参数修改 Hook

```typescript
"chat.params"?: (
  input: { sessionID: string; agent: string; model: Model; provider: ProviderContext; message: UserMessage },
  output: { temperature: number; topP: number; topK: number; options: Record<string, any> }
) => Promise<void>
```

修改 LLM 调用的参数（温度、topP 等）。

#### 工具执行拦截 Hook

```typescript
"tool.execute.before"?: (
  input: { tool: string; sessionID: string; callID: string },
  output: { args: any }
) => Promise<void>

"tool.execute.after"?: (
  input: { tool: string; sessionID: string; callID: string; args: any },
  output: { title: string; output: string; metadata: any }
) => Promise<void>
```

在工具执行前后进行拦截和修改。这是实现多 agent 调度的关键 Hook。

#### Shell 环境变量 Hook

```typescript
"shell.env"?: (
  input: { cwd: string; sessionID?: string; callID?: string },
  output: { env: Record<string, string> }
) => Promise<void>
```

注入自定义环境变量。

#### 权限请求 Hook

```typescript
"permission.ask"?: (
  input: Permission,
  output: { status: "ask" | "deny" | "allow" }
) => Promise<void>
```

控制权限请求行为。

#### 命令执行 Hook

```typescript
"command.execute.before"?: (
  input: { command: string; sessionID: string; arguments: string },
  output: { parts: Part[] }
) => Promise<void>
```

拦截和处理自定义命令（如 `/interview`、`/preset` 等）。

### 3.2 实验性 Hook

#### 消息列表转换

```typescript
"experimental.chat.messages.transform"?: (
  input: {},
  output: { messages: { info: Message; parts: Part[] }[] }
) => Promise<void>
```

在消息发送给 LLM 前进行转换。

#### 系统提示转换

```typescript
"experimental.chat.system.transform"?: (
  input: { sessionID?: string; model: Model },
  output: { system: string[] }
) => Promise<void>
```

注入系统提示词（System Prompt），这是实现 orchestrator 调度的关键 Hook。

#### 会话压缩 Hook

```typescript
"experimental.session.compacting"?: (
  input: { sessionID: string },
  output: { context: string[]; prompt?: string }
) => Promise<void>
```

自定义会话压缩逻辑。

#### 文本补全 Hook

```typescript
"experimental.text.complete"?: (
  input: { sessionID: string; messageID: string; partID: string },
  output: { text: string }
) => Promise<void>
```

自定义文本补全。

### 3.3 可用事件类型

| 类别 | 事件 |
|------|------|
| **Session** | `session.created`, `session.updated`, `session.deleted`, `session.idle`, `session.error`, `session.compacted`, `session.status`, `session.diff` |
| **Message** | `message.updated`, `message.removed`, `message.part.updated`, `message.part.removed` |
| **Tool** | `tool.execute.before`, `tool.execute.after` |
| **File** | `file.edited`, `file.watcher.updated` |
| **Permission** | `permission.asked`, `permission.replied` |
| **Shell** | `shell.env` |
| **TUI** | `tui.toast.show`, `tui.prompt.append`, `tui.command.execute` |
| **Command** | `command.executed` |
| **LSP** | `lsp.updated`, `lsp.client.diagnostics` |
| **Server** | `server.connected` |
| **Installation** | `installation.updated` |
| **Todo** | `todo.updated` |

### 3.4 Hook 触发机制

所有 Hook 通过统一的 `Plugin.trigger()` 函数触发：

```typescript
export async function trigger<Name extends keyof Hooks>(
  name: Name,
  input: Input,
  output: Output
): Promise<Output> {
  if (!name) return output
  for (const hook of await state().then((x) => x.hooks)) {
    const fn = hook[name]
    if (!fn) continue
    await fn(input, output)  // 顺序执行，每个插件可以修改 output
  }
  return output
}
```

## 四、插件配置方式

### 4.1 从本地文件加载

将插件文件放在 `.opencode/plugins/`（项目级）或 `~/.config/opencode/plugins/`（全局级）。

### 4.2 从 npm 加载

在 `opencode.json` 中配置：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "plugin": ["opencode-helicone-session", "opencode-wakatime", "@my-org/custom-plugin"]
}
```

### 4.3 本地插件使用 npm 依赖

在 `.opencode/` 目录下创建 `package.json`：

```json
{
  "dependencies": {
    "shescape": "^1.0.0"
  }
}
```

OpenCode 启动时会自动运行 `bun install` 安装依赖。

## 五、发布流程

### 5.1 发布前检查清单

- [ ] 包名使用 kebab-case，建议前缀 `opencode-`
- [ ] `package.json` 包含 `"type": "module"`
- [ ] 包含 `"peerDependencies": { "@opencode-ai/plugin": "^1.0.0" }`
- [ ] 包含 `"keywords": ["opencode-plugin"]`
- [ ] `main` 和 `types` 指向正确的构建输出
- [ ] 构建产生 `.js` 和 `.d.ts` 文件
- [ ] 没有顶层副作用（网络请求、文件写入）
- [ ] README 文档完整

### 5.2 构建和发布

```bash
# 构建
npm run build

# 验证模块加载
node -e "import('./dist/index.js').then(console.log)"

# 登录 npm
npm login

# 发布
npm publish --access public
```

### 5.3 用户安装方式

```json
{
  "plugin": ["opencode-my-plugin@0.1.0"]
}
```

### 5.4 自动化发布（GitHub Actions）

```yaml
name: Publish
on:
  push:
    tags: ['v*']
jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v1
      - run: bun install
      - run: bun run build
      - run: npm publish --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```

## 六、示例代码片段

### 6.1 .env 保护插件

```javascript
export const EnvProtection = async ({ project, client, $, directory, worktree }) => {
  return {
    "tool.execute.before": async (input, output) => {
      if (input.tool === "read" && output.args.filePath.includes(".env")) {
        throw new Error("Do not read .env files")
      }
    },
  }
}
```

### 6.2 会话通知插件

```javascript
export const NotificationPlugin = async ({ project, client, $, directory, worktree }) => {
  return {
    event: async ({ event }) => {
      if (event.type === "session.idle") {
        await $`osascript -e 'display notification "Session completed!" with title "opencode"'`
      }
    },
  }
}
```

### 6.3 自定义工具插件

```typescript
import { type Plugin, tool } from "@opencode-ai/plugin"

export const CustomToolsPlugin: Plugin = async (ctx) => {
  return {
    tool: {
      project_stats: tool({
        description: "Get project statistics",
        args: {},
        async execute(args, context) {
          const { directory, worktree } = context
          return `Project: ${directory}`
        },
      }),
    },
  }
}
```

### 6.4 环境变量注入插件

```typescript
export const EnvInjector: Plugin = async (ctx) => {
  return {
    "shell.env": async (input, output) => {
      output.env.MY_CUSTOM_VAR = "value"
      output.env.PATH = `/custom/path:${output.env.PATH}`
    },
  }
}
```

### 6.5 完整的工具拦截和修改插件

```typescript
import { escape } from "shescape"

export const SecurityPlugin: Plugin = async (ctx) => {
  return {
    "tool.execute.before": async (input, output) => {
      // 阻止危险命令
      if (input.tool === "bash" && output.args.command.includes("rm -rf")) {
        throw new Error("Dangerous command blocked")
      }
      
      // 转义 bash 命令
      if (input.tool === "bash") {
        output.args.command = escape(output.args.command)
      }
    },
    
    "tool.execute.after": async (input, output) => {
      // 记录工具执行结果
      console.log(`Tool ${input.tool} executed: ${output.title}`)
    },
  }
}
```

## 七、调试技巧

1. **插件未加载**：检查 TypeScript 语法错误，使用 `opencode --verbose`
2. **Hook 未触发**：验证 Hook 名称拼写（区分大小写）
3. **状态未持久化**：使用 session-keyed Maps，不要用全局变量
4. **依赖未找到**：在 `.opencode/` 目录创建 `package.json` 并运行 `bun install`
5. **使用 console.log**：在 Hook 中添加日志输出进行调试
